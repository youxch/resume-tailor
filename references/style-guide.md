# 写作风格指南
# 来源：从"自己简历的迭代版本" + "一份认可的范本简历"中提炼出的写作规律
# （这是一份示例性整理，建议你也整理一份自己的版本——把你认可的简历范本拆解成可复用的格式规则）

---

## 一、节顺序（重要）

你的简历节顺序（优先参考）：
```
科研项目（第一，最重要）
教育背景
专业技能
个人荣誉
发表论文
```

范本简历（投递某个方向时参考）：
```
求职意向（明确一句话：天线/射频/微波技术研究、开发）
教育背景（含GPA）
科研项目
专业技能
个人荣誉
发表论文
```

**差距提示：** 你的当前简历没有"求职意向"这一行。建议加上，尤其投递天线/RF 岗位时。

---

## 二、项目描述的固定 4 段式结构

两份简历完全一致，每个项目必须写 4 块：

```
1. 研究目标（一段话）
   格式：针对【传统方法的瓶颈】，首次提出【你的创新范式/方法】，极大【效果】。

2. 研究内容（编号列举）
   格式：①… ②… ③… ④…（用圆圈数字，不用减号bullet）

3. 关键结果（标签式）
   格式：「子任务名称：」+说明句，关键数字加粗/用黑体
   例：数据筛选：首次提出小规模数据筛选方法（无需物理知识，仅利用统计分布），所需数据规模降低至原有的**0.012%**，仿真成本降低**99%**。

4. 研究成果（一句话汇总）
   格式：发表【级别】期刊/会议论文X篇，获【奖项】，相关成果【GitHub/专利等补充说明】。
```

---

## 三、创新标注规则

**必须用"首次"标注所有创新点：**
- 行业内首个高性能并行仿真系统
- 首次提出小规模数据筛选方法
- 首次提出RGB三色法
- 首次提出光控微波开关

**对比句式（范本风格，天线方向强化使用）：**
> 区别于【传统方法/已有方案】，首次提出【你的方案】，实现/突破【量化指标】。

---

## 四、数字格式规范

| 类型 | 写法 |
|------|------|
| 倍数加速 | 效率提升300倍；加快5000倍 |
| 百分比 | 数据规模降低至原有的0.012%；仿真成本降低99% |
| 时间缩短 | 时间缩短至1秒；设计时间缩短至5分钟以内 |
| 相对比较 | 相比传统人为收集，效率提升300倍 |
| 超额完成 | 超出原定指标XX%（如有合同/课题验收要求时适用） |
| 工程规格 | 增益大于20 dBi；±60°波束扫描（范本风格，精确工程数字） |

---

## 五、专业技能写法

**不是简单罗列技能，而是"能力宣言"式段落：**

```
精通AI4S（AI for Science）研究范式：掌握多学科知识，技能储备包括（1）…（2）…（3）…
优秀的跨学科沟通与协作能力：非常擅长在跨学科团队中主导项目，具备向不同背景的协作者清晰传达技术方案的能力。
优秀的数理基础：擅于利用量化模型对问题重新建模，熟悉各种优化算法（粒子群、遗传、代理模型等）。
优秀的语言能力：优秀的中英文沟通、写作能力，国际交流与汇报经验丰富（多次获奖）。
```

**天线/RF方向时（范本风格）更具体化：**
- 精通高效率、高精准的全波电磁仿真（如CST、HFSS、FEKO、COMSOL）
- 精通有源与无源的天线与阵列的近、远场测试
- 精通单层、多层印制电路板绘制、工艺设计
- 熟练使用网分、示波器、信号源、频谱仪等

## 六、论文格式

**标准格式：**
```
作者（一作加粗），其他作者, "标题," 期刊/会议名称缩写, vol.XX, no.X, pp.XXXX, 月份 年份. (括注：级别+角色)
```

**括注优先级（按重要性排序写）：**
1. 期刊级别：中科院1区TOP / JCR1区
2. 作者角色：第一作者，唯一学生作者
3. 特殊标注：受邀报告（Invited Talk）/ 热点论文 / GitHub领域首篇
4. 获奖：Wiley高被引奖 / Marina Forum学生竞赛提名奖

**状态标注：**
- 已接受：accept.
- 在准备：manuscript in preparation.

---

## 七、动词选用规范

| 用 | 避免用 |
|----|-------|
| 首次提出 / 提出 | 负责 / 完成 |
| 构建 / 搭建 | 参与 / 协助 |
| 主导 / 主持 | 涉及 / 了解 |
| 实现 / 突破 | 做了 / 进行了 |
| 验证 / 制备 | 基本完成 |

---

## 八、无主语原则

全文不出现"我"/"本人"，所有句子以动词或名词开头。

**错误：** 我提出了一种新方法
**正确：** 首次提出一种新方法，实现…

---

## 九、中英混排规范（两份简历均遵循）

- 英文期刊/会议名缩写用斜体（Word里设置，Markdown用*斜体*）
- 技术术语：中文表述在前，英文缩写在括号内：特征模式分析(CMA)
- GitHub、Wiley、CLIP 这类专有名词保持英文大小写
- 数字用阿拉伯数字，不用中文数字：300倍，不是三百倍
