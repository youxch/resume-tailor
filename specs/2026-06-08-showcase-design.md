# resume-tailor 展示页 + 独立开源仓库 — 设计方案

日期：2026-06-08

## 背景

尤栖冲写了一篇小红书读博日记，介绍自己开发的 Claude skill「resume-tailor」（粘贴JD自动分析匹配度、改写简历语言、推荐岗位），文末提到"已开源，GitHub搜 youxch/resume-tailor"。
现状：该 skill 目前是 `Resume_auto`（个人简历仓库）下的一个未跟踪子目录 `resume-tailor/`，其中：
- `SKILL.md` 含真实姓名、目标公司列表、指向私人仓库的部署子流程
- `references/candidate-profile.md`、`references/project-details.md` 含真实 PII（邮箱、电话、出生年月、项目细节）
- `references/style-guide.md`、`evals/evals.json` 是通用方法论，不含个人敏感信息

需要做两件事：
1. 把 skill 整理成可公开的独立仓库 `youxch/resume-tailor`，剥离个人敏感信息
2. 做一个干净整洁的展示页（HTML），既能截图发小红书，也能部署成该仓库的 GitHub Pages 主页

## 展示页设计

### 叙事与结构

故事线：**问题 → 方法 → 效果**（与小红书文案节奏一致）
- 问题：换赛道找工作，背景过硬但HR看不出关联，简历投出去石沉大海
- 方法：粘贴JD → AI多维度打分匹配度 → 指出优势/差距 → 自动改写项目描述和关键句
- 效果：命中率变化（用虚构示例展示"改写前/后"对比，不使用真实项目细节）

### 版式（已用可视化协作工具与用户确认：方案 B）

左右分栏 / 杂志式布局：
- 左侧：固定标题区（项目名 `resume-tailor` + 一句话定位）+ 锚点导航（痛点 / 怎么用 / 效果 / GitHub）
- 右侧：可滚动正文区，按锚点顺序排列对应内容区块

这个布局信息密度适中、"工具感"强，同时单屏截图也好看，适合一图发小红书。

### 视觉风格（已用可视化协作工具与用户确认：冷白极简学术风）

| 元素 | 取值 |
|---|---|
| 背景 | 纯白 `#ffffff`，侧栏可用极浅灰 `#fafbfc` 区分 |
| 正文色 | 深灰 `#1f2328` |
| 次要文字 | 中灰 `#6b7280` |
| 强调色 | 靛蓝 `#3060e0`（用于锚点高亮、按钮、链接、关键数字） |
| 分隔线 | 极浅灰 `#e3e5e8` |
| 正文字体 | Inter, sans-serif |
| 代码/标签字体 | IBM Plex Mono, monospace |

整体气质：理工科、严谨、留白充分，符合"干净整洁"的要求。

### 内容大纲（右侧滚动区，按锚点分块）

1. **Hero**：`resume-tailor` + 一句话定位（"针对JD自动分析匹配度、改写简历语言的 Claude Skill"）+ GitHub 链接按钮
2. **痛点**：换赛道求职的困境（背景硬但简历语言对不上岗位语言，内推圈子用不上）——呼应小红书原文的真实感受，但不涉及任何具体公司/项目细节
3. **怎么用**：3 步流程图式排版
   ① 粘贴目标岗位 JD
   ② Skill 从HR/技术面试官/产品负责人三个视角打分，指出优势与差距
   ③ 自动改写项目描述、关键句，按打分结果迭代到高分再输出
4. **Before / After 示例**（虚构素材，非真实项目）：左右对照卡片，展示"改写前的平铺直叙描述" vs "改写后命中JD关键词、带量化结果的版本"，体现"语言翻译"这件事的价值
5. **效果与开源**：简述命中率变化的主观感受（不编造具体数字），GitHub 仓库链接、安装/使用方式简述、开源协议
6. **Footer**：作者署名（姓名 + GitHub ID，公开信息），无联系方式

### 不会出现在页面上的内容
- 邮箱、电话、出生年月等联系方式/隐私信息
- 任何真实目标公司名单或真实项目技术细节
- 真实的简历改写片段（一律用虚构示例）

## GitHub 仓库迁移与脱敏方案

### 仓库结构

新建独立仓库 `youxch/resume-tailor`（与含个人简历内容的 `Resume_auto` 完全分离），结构：

```
resume-tailor/
├── index.html              # 展示页（GitHub Pages 自动识别为主页）
├── SKILL.md                # 通用化改写后的 skill 定义
├── references/
│   ├── candidate-profile.template.md   # 模板化示例（占位符，无真实信息）
│   ├── project-details.template.md     # 模板化示例（占位符，无真实信息）
│   └── style-guide.md                  # 原样保留（通用方法论）
├── evals/
│   └── evals.json          # 原样保留（通用方法论）
└── README.md               # 项目说明 + 安装/使用方法 + 指向 index.html 的预览链接
```

### SKILL.md 通用化改写要点

- 把"尤栖冲"等真实姓名替换为 `[候选人姓名]` 占位符，描述改为"这是一个可定制的简历定制 skill 模板，需要你提供自己的候选人档案"
- 删除/替换"快速参考：常见目标公司方向"表格中的真实公司列表 → 改写成"如何根据你自己的目标行业建立这张表"的方法说明
- 删除"GitHub 部署子流程"一节中指向私人 `Resume_auto` 仓库的具体路径和命令 → 改写成通用的"如何把你的网页版简历部署到 GitHub Pages"的步骤说明（不含任何指向用户私人仓库的具体信息）
- 保留核心方法论不动：方向判断逻辑、四段式项目结构模板、受众视角闭环评审系统（这是这个 skill 真正的价值所在，且本身不含个人信息）

### references 文件处理

- `candidate-profile.md`、`project-details.md` → 改名为 `*.template.md`，内容替换为字段说明 + 占位符示例（例如用"某高校博士生""某 AI 辅助设计项目"这类通用占位描述展示该填什么、格式是什么），让使用者一看就知道该怎么填自己的真实信息
- `style-guide.md`、`evals/evals.json` 原样复制（已确认是通用方法论，不含 PII）

### 部署方式

- `index.html` 放在仓库根目录，GitHub Pages 设置 `Deploy from branch: main / (root)`
- 访问地址：`https://youxch.github.io/resume-tailor/`
- 同一个 `index.html` 也可以本地浏览器打开直接截图发小红书
