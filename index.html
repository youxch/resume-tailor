Claude Code Skill

痛点

换赛道找工作，简历投出去就没了下文

背景过硬，但目标岗位跟原本的方向不在一个圈子里——师兄内推、导师推荐这些路子突然都用不上了。最难的地方不是能力不够，而是 HR 和面试官看不出你的经历和这个岗位之间的关系，需要有人帮你把"你做过什么"翻译成"这个岗位想要什么"。

怎么用

把 JD 粘进去，剩下的交给它

01
粘贴目标岗位 JD不需要额外整理，原文直接丢进去
02
三视角打分，定位差距分别从 HR / 技术面试官 / 产品负责人 视角给匹配度打分，标出优势和短板
03
自动改写，迭代到高分重写项目描述与关键句，按打分结果反复修改，直到三个视角都达标再输出

改写示例（演示用虚构素材）

同一段经历，换一种"语言"

改写前

负责某检测算法的设计与实现，完成了模型训练和性能测试，撰写了相关技术文档。

改写后

针对某场景下传统方法误检率高的问题，提出一种轻量化检测算法，在无需额外标注数据的前提下首次实现端到端推理，准确率提升 18%，已在内部完成验证并产出可复用工具链。

效果与开源

命中率明显不一样了

用改写后的版本投出去，"发出去就没消息"的情况比之前少了很多——HR 能更快看懂你和岗位的关联，回复速度也更快。这个 skill 已经开源，安装到 Claude Code / Claude Desktop 后即可使用，欢迎按自己的背景定制 references 里的素材文件。
